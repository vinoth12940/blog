LiteParse: The Open-Source Document Parser That Runs Entirely on Your Machine

LlamaParse is being deprecated. LiteParse is its free, local replacement — no API keys, no cloud, no LLMs required. With spatial layout, bounding boxes, and built-in OCR.

April 9, 2026 · Document Parsing / Open Source

LiteParse extracts structured text from documents locally, with spatial layout and bounding boxes

Every developer building RAG pipelines has hit the same wall — your PDF parser either costs money, sends your documents to a cloud API, or produces garbage output. LlamaParse, LlamaIndex's flagship cloud document parser, is now being deprecated (May 1, 2026), and users are being migrated to llama-cloud which still requires API keys and per-page pricing.

Enter LiteParse — LlamaIndex's fully open-source, locally-running answer to document parsing. No API keys. No cloud calls. No LLMs needed. Just fast, accurate PDF parsing with spatial layout, bounding boxes, built-in OCR, and multi-format support. It has already racked up 4.1k GitHub stars and 324+ commits with active development.

Here's everything you need to know about LiteParse, with real code examples in both TypeScript and Python.

Why LiteParse?

LlamaParse (the cloud version) was great at what it did — but it required an API key and sent your documents through LlamaIndex's servers. For privacy-conscious applications, air-gapped environments, or just keeping costs at zero, that's a dealbreaker.

LiteParse represents a fundamentally different philosophy — everything runs on your machine. Here's what it brings to the table:

  • Zero cloud dependencies — your documents never leave your machine
  • Spatial text parsing — text comes back positioned where it appears on the page
  • Bounding boxes for every text element — perfect for visual citations in RAG
  • Built-in OCR via Tesseract.js, with pluggable external OCR servers (EasyOCR, PaddleOCR)
  • Multi-format support — PDF, DOCX, XLSX, PPTX, images, and more
  • Page screenshots — render PDF pages as images for LLM vision workflows
  • Agent skill integration — works as a Claude Code or Cursor skill out of the box
  • Available in TypeScript and Python
  • Apache 2.0 licensed
LiteParse vs LlamaParse Cloud at a Glance
  • LiteParse: Open-source, local-only, free, Apache 2.0 license, spatial layout + bounding boxes, no API key
  • LlamaParse Cloud: Cloud API, per-page pricing ($0.003/page after free tier), LLM-powered multimodal parsing, custom prompts, managed infra

Getting Started

Installation

Install globally via npm:

npm i -g @llamaindex/liteparse

Or via Homebrew (macOS and Linux):

brew tap run-llama/liteparse
brew install llamaindex-liteparse

Quick CLI Demo

Once installed, parsing a document is a single command:

# Parse a PDF and print text to stdout
lit parse document.pdf

# Save output to a file
lit parse document.pdf -o output.txt

# Get structured JSON with bounding boxes
lit parse document.pdf --format json -o output.json

# Parse only specific pages
lit parse document.pdf --target-pages "1-5,10,15-20"

Batch Parsing

Process an entire directory at once:

lit batch-parse ./pdfs ./outputs

Page Screenshots

Generate page images for LLM vision workflows:

lit screenshot document.pdf -o ./screenshots

TypeScript: Library Usage

The CLI is great for quick tasks, but the real power comes from using LiteParse programmatically.

npm install @llamaindex/liteparse

Basic Parsing

import { LiteParse } from "@llamaindex/liteparse";

const parser = new LiteParse({ ocrEnabled: true });
const result = await parser.parse("document.pdf");

// Full document text with layout preserved
console.log(result.text);

// Per-page data
for (const page of result.pages) {
  console.log(`Page ${page.pageNum}: ${page.textItems.length} text items`);
}

JSON Output with Bounding Boxes

This is where LiteParse really shines — every text element comes with spatial coordinates (x, y, width, height) in PDF points (1 point = 1/72 inch), with the origin at the top-left corner.

const parser = new LiteParse({ outputFormat: "json" });
const result = await parser.parse("document.pdf");

for (const page of result.json?.pages || []) {
  for (const item of page.textItems) {
    console.log(
      `[${item.x}, ${item.y}] → [${item.x + item.width}, ${item.y + item.height}] ${item.text}`
    );
  }
}

Full Configuration

const parser = new LiteParse({
  ocrEnabled: true,
  ocrServerUrl: "http://localhost:8828/ocr",
  ocrLanguage: "fra",
  dpi: 300,
  outputFormat: "json",
  targetPages: "1-10",
  password: "secret", // for encrypted/protected documents
});

Parse from Memory (Zero Disk I/O)

You can pass raw bytes directly. For PDFs, this means zero disk I/O — no temp files written:

import { readFile } from "fs/promises";

const parser = new LiteParse();

// From a file read
const pdfBytes = await readFile("document.pdf");
const result = await parser.parse(pdfBytes);

// From an HTTP response
const response = await fetch("https://example.com/document.pdf");
const buffer = Buffer.from(await response.arrayBuffer());
const result2 = await parser.parse(buffer);

Generate Page Screenshots

Useful for sending PDF pages directly to LLMs for vision-based analysis:

const parser = new LiteParse();
const screenshots = await parser.screenshot("document.pdf");

for (const shot of screenshots) {
  console.log(`Page ${shot.pageNum}: ${shot.width}x${shot.height}`);
  // shot.imageBuffer contains the raw PNG/JPG data
}

// Parse specific pages only from a buffer
const pdfBytes = await readFile("document.pdf");
const shots = await parser.screenshot(pdfBytes, [1, 2, 3]);

Python: Library Usage

The Python package (pip install liteparse, version 1.2.1) wraps the Node.js CLI under the hood. Node.js (>= 18) must be installed, and Python >= 3.10 is required.

npm install -g @llamaindex/liteparse  # CLI first
pip install liteparse

Basic Parsing

from liteparse import LiteParse

parser = LiteParse()
result = parser.parse("document.pdf")

# Full document text
print(result.text)

# Per-page data
for page in result.pages:
    print(f"Page {page.pageNum}: {len(page.textItems)} text items")

Full Configuration

result = parser.parse(
    "document.pdf",
    ocr_enabled=True,
    ocr_server_url="http://localhost:8828/ocr",
    ocr_language="fra",
    dpi=300,
    target_pages="1-5",
    password="secret",
)

Parse from Bytes

with open("document.pdf", "rb") as f:
    pdf_bytes = f.read()

result = parser.parse(pdf_bytes)
print(result.text)

Batch Parsing (Faster for Multiple Files)

Batch mode reuses the PDF engine and is significantly faster than parsing files one at a time:

result = parser.batch_parse(
    input_dir="./documents",
    output_dir="./output",
    recursive=True,
    extension_filter=".pdf",
)

print(f"Output written to: {result.output_dir}")

Bounding Boxes and Visual Citations

One of LiteParse's most powerful features for RAG systems is spatial awareness. Every text element has coordinates, which means you can show users exactly where in a document a piece of information came from.

Converting PDF Points to Image Pixels

Text coordinates are in PDF points, but screenshots are in pixels. You need to scale by DPI/72:

const DPI = 150;
const SCALE = DPI / 72; // ~2.08x

function itemToPixels(item, dpi = 150) {
  const scale = dpi / 72;
  return {
    x: item.x * scale,
    y: item.y * scale,
    width: item.width * scale,
    height: item.height * scale,
  };
}

Searching for Phrases with searchItems

A single phrase like "0°C to 70°C" may span multiple adjacent text items. LiteParse provides a searchItems utility that concatenates consecutive items, finds matches, and returns merged text items with combined bounding boxes:

import { LiteParse, searchItems } from "@llamaindex/liteparse";

const parser = new LiteParse({ outputFormat: "json" });
const result = await parser.parse("report.pdf");

for (const page of result.json.pages) {
  const matches = searchItems(page.textItems, {
    phrase: "0°C to 70°C"
  });
  for (const match of matches) {
    console.log(
      `Found "${match.text}" at (${match.x}, ${match.y}) ${match.width}×${match.height}`
    );
  }
}
Use Case: RAG with Visual Citations

Parse a document, find relevant text with an LLM, then use the bounding boxes to highlight the exact region on a page screenshot. This gives users verifiable proof of where information was sourced from — critical for legal, medical, and financial document AI.

OCR: Built-in and Pluggable

LiteParse uses OCR selectively — only on embedded images or pages where native text extraction didn't find text. This keeps parsing fast while still capturing text from scanned documents.

Built-in Tesseract.js (Bundled, No Extra Install)

# English (default)
lit parse document.pdf

# French
lit parse document.pdf --ocr-language fra

# Japanese
lit parse document.pdf --ocr-language jpn

# Disable OCR entirely for speed on native-text PDFs
lit parse document.pdf --no-ocr

External OCR Servers (GPU-Accelerated)

For higher accuracy or GPU acceleration, point LiteParse at an HTTP OCR server:

# EasyOCR server
git clone https://github.com/run-llama/liteparse.git
cd liteparse/ocr/easyocr
pip install -r requirements.txt
python server.py

# Parse with EasyOCR
lit parse document.pdf --ocr-server-url http://localhost:8828/ocr

PaddleOCR works identically — start its server and point LiteParse at the same endpoint.

Custom OCR Servers

Integrate any OCR engine by implementing a simple HTTP API:

  • Endpoint: POST /ocr
  • Request: multipart form with file (image binary) and optional language (ISO 639-1)
  • Response: {"results": [{"text": "...", "bbox": [x1, y1, x2, y2], "confidence": 0.95}]}

Multi-Format Support

LiteParse handles more than just PDFs. It auto-converts other formats to PDF using system tools, then runs the same parsing pipeline.

  • Office documents (via LibreOffice): .doc, .docx, .docm, .ppt, .pptx, .pptm, .xls, .xlsx, .xlsm, .csv, .tsv, .rtf, .odt, .odp, .ods
  • Images (via ImageMagick): .jpg, .jpeg, .png, .gif, .bmp, .tiff, .webp, .svg

Install the required system dependency, then parse anything:

# Install LibreOffice for Office docs
brew install --cask libreoffice  # macOS
apt-get install libreoffice       # Ubuntu

# Install ImageMagick for images
brew install imagemagick           # macOS
apt-get install imagemagick        # Ubuntu

# Then parse anything
lit parse report.docx
lit parse slides.pptx --format json
lit parse scan.png

Agent Integration: Claude Code and Cursor

LiteParse ships as a coding agent skill via Vercel's skills utility. Once installed, your AI coding agent can parse documents, generate screenshots, and extract text on the fly:

npx skills add run-llama/llamaparse-agent-skills --skill liteparse

After installation, you can prompt your agent with natural language:

  • "Parse this PDF and extract the text as JSON"
  • "Extract text from all the DOCX files in the ./contracts folder"
  • "Screenshot pages 1-5 of this PDF at 300 DPI"
  • "Get the bounding boxes for all text on page 3"

Default behavior can be configured via a liteparse.config.json file in your project root:

{
  "ocrLanguage": "en",
  "ocrEnabled": true,
  "maxPages": 1000,
  "dpi": 150,
  "outputFormat": "json"
}

When to Use LiteParse vs LlamaParse Cloud

Decision Framework

Choose LiteParse when:

  • You need local/air-gapped processing (no cloud allowed)
  • You want zero recurring costs
  • You need spatial layout and bounding boxes for RAG citations
  • You're building agents that parse documents on the fly
  • You process small-to-medium document volumes

Choose LlamaParse Cloud (llama-cloud) when:

  • You need multimodal parsing with vision LLMs for complex layouts
  • You need custom parsing prompts that require LLM understanding
  • You handle massive document volumes at enterprise scale
  • You want managed infrastructure with no ops overhead

Environment Variables for Production

For Docker containers, CI/CD pipelines, or air-gapped deployments:

  • TESSDATA_PREFIX — Path to Tesseract .traineddata files for offline environments
  • LITEPARSE_TMPDIR — Override temp directory for format conversion (defaults to system tmp)

Wrapping Up

LiteParse fills a gap that's been growing in the document parsing ecosystem — a fast, free, local parser that doesn't sacrifice quality. The combination of spatial layout parsing, bounding boxes, and pluggable OCR makes it particularly compelling for RAG systems where knowing where text came from matters as much as the text itself.

With 4.1k stars, Apache 2.0 licensing, and active development, LlamaIndex is clearly investing in LiteParse as the local-first complement to their cloud offering. If you're building document-heavy AI applications, it's worth a serious look.

Install it, parse a document, and see if the bounding box output changes how you think about document processing.
