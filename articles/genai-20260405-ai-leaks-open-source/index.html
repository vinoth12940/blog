The Week AI Broke Wide Open: Claude Code Leaked and Gemma 4 Went Truly Free

In a span of just 72 hours, the AI industry experienced two seismic events. Anthropic accidentally shipped the complete source code of its Claude Code AI coding assistant in a production release. Less than 24 hours later, Google dropped Gemma 4 under an Apache 2.0 license — the most capable truly open-source model family ever released.

One was a catastrophic security blunder. The other was a deliberate open-source statement. Together, they tell a story about where AI is headed in 2026 — and the tension between closed commercial products and the open-source movement that's reshaping the industry.

The 72 Hours That Shook AI

Here's what happened, hour by hour:

  • March 31, 2026 (evening): Anthropic releases Claude Code v2.1.88 to npm. Buried inside the package is a 59.8MB file containing the complete, unminified source code of Claude Code — the AI-powered coding tool used by thousands of developers.
  • April 1, morning: A developer spots the exposed source map file and posts it on X. Within hours, the code is forked and reposted across GitHub.
  • April 1, evening: Anthropic files a DMCA takedown notice targeting the leaked repository and its forks. GitHub's enforcement sweeps up roughly 8,100 repositories — including legitimate forks of Anthropic's own public Claude Code repo.
  • April 1, late night: Anthropic retracts the mass takedown, narrowing it to just one repository and 96 fork URLs. The damage is done — copies are already everywhere.
  • April 2: Google announces Gemma 4, a family of four open models under the Apache 2.0 license — no restrictions, no custom terms. The 31B model debuts at #3 on the open-source Arena leaderboard.

The timing was almost poetic. As Anthropic scrambled to lock down proprietary code, Google was busy giving its best models away.

How Claude Code's Source Got Out

The leak wasn't sophisticated. It wasn't a hack, an insider threat, or a supply chain attack. It was a plain developer error — the kind that happens when you're shipping fast.

Anthropic's Claude Code is distributed as an npm package. When version 2.1.88 was built for production, a source map file was included in the bundle. Source maps are supposed to help with debugging by mapping minified code back to the original source. But this one contained the entire unminified source tree.

"The source is, for all practical purposes, permanently public." — Systima, AI compliance consultancy

A developer using the handle Fried_rice on X was the first to publicly flag the exposed file. Boris Cherny, a Claude Code engineer at Anthropic, later confirmed it was human error: "plain developer error, not a tooling bug."

The version was quickly pulled from npm, but not before developers had already cloned, forked, and analyzed the codebase across GitHub.

What Developers Found Inside

Once the code was public, the AI developer community did what it does best — tore it apart. Here's what was discovered:

  • Anti-cloning protections: Claude Code includes techniques specifically designed to prevent bad actors from reverse-engineering or forking the tool. Ironically, the leak itself exposed these protections.
  • AI output laundering: A mode that strips evidence of AI authorship from generated code. When Anthropic employees use Claude Code to contribute to open-source projects, the tool removes traces that an AI wrote the commits. This has implications for transparency in AI-assisted development.
  • MAGIC DOC headers: Anthropic engineers use special file headers internally to structure AI-assisted workflows. This pattern, discovered in the leaked code, has already been adopted by some developers as a useful convention.
  • Prompt caching pipeline: The internal implementation of Anthropic's prompt caching system, which Claude Code uses to deliver faster responses by reusing context across tool calls.
One independent security researcher estimated that a determined team could bypass Claude Code's anti-cloning mechanisms within an hour using a MITM proxy or by setting the CLAUDE_CODE_DISABLE_EXPERIMENTAL_BETAS environment variable.

The DMCA Disaster: Collateral Damage at Scale

Anthropic's response to the leak was arguably worse than the leak itself.

On the evening of April 1, Anthropic filed a DMCA takedown notice with GitHub targeting the repository hosting the leaked code and its forks. The problem? The leaked repository was part of a fork network connected to Anthropic's own public Claude Code repository. GitHub's enforcement mechanism treated the entire fork network as a single entity.

The result: approximately 8,100 repositories were taken down, including legitimate forks that developers had created for their own contributions to Claude Code's open components.

Anthropic quickly realized the scope of the damage and filed a retraction, but the message was clear: DMCA is a blunt instrument, and using it in a panic can cause more harm than good.

The incident raises serious questions about how AI companies should handle source code leaks in an era where code spreads faster than legal notices.

Meanwhile, Google Made Everything Free

While Anthropic was dealing with its leak, Google dropped Gemma 4 — and the details matter.

Gemma 4 comes in four sizes, each targeting different deployment scenarios:

  • Gemma 4 E2B (Effective 2B): Runs on mobile devices and edge hardware. 128K context window. Built for on-device AI experiences.
  • Gemma 4 E4B (Effective 4B): Capable of running on a Raspberry Pi. 128K context. Proves that serious AI doesn't need serious hardware.
  • Gemma 4 26B MoE: Mixture-of-Experts model that only activates 3.8 billion parameters during inference. Fast token generation with 256K context window.
  • Gemma 4 31B Dense: The flagship. Debuts at #3 on the Arena open-source leaderboard (ELO ~1452), behind only GLM-5 and Kimi 2.5. Outperforms models 20x its size on many benchmarks.

But the real headline isn't the performance — it's the license.

Previous Gemma generations shipped under a custom Google license that many developers found restrictive. Gemma 4 switches to Apache 2.0 — the gold standard for open-source software. No usage restrictions. No commercial limitations. No fine print.

Every model supports multimodal input (images and video) out of the box. They all support function calling and structured output for building agentic systems. And they're available on Hugging Face, Kaggle, and Ollama — download them and run them locally.

Gemma as a family has been downloaded over 400 million times since the first release in February 2024. The "Gemmaverse" community has created over 100,000 variants.

The Bigger Picture: AI on Your Phone

Google releasing E2B and E4B models isn't just about developers — it's about smartphones. The fact that Gemma 4 E2B can run on-device means Google is positioning itself for a future where your phone runs its own AI model without calling the cloud.

This is where the industry is heading. Apple, Google, and Samsung are all building NPUs (Neural Processing Units) into their chips. Models like Gemma 4 E2B are the software that will make those chips useful.

What This Week Means for Developers

The juxtaposition of these two events — one accidental, one deliberate — reveals a lot about the current state of AI:

  • Open source doesn't mean open code. Claude Code was "open to use" but entirely closed-source. When it leaked, we learned just how much Anthropic was hiding behind the npm package. The anti-cloning protections and AI output laundering features were never going to be visible otherwise.
  • DMCA is a blunt instrument for the GitHub era. Taking down 8,100 repos to stop a leak is like burning down a neighborhood to catch a burglar. Anthropic's retraction was necessary but the damage to developer trust was already done.
  • Apache 2.0 is the new open-source standard. Google's move from a custom license to Apache 2.0 for Gemma 4 is a signal. Developers have made it clear that permissive licensing matters. Companies that want community adoption need to actually commit to open source, not just perform it.
  • On-device AI is no longer experimental. A model that runs on a Raspberry Pi and processes images and video is not a demo — it's a product. Expect to see Gemma 4 E2B and E4B showing up in apps, devices, and embedded systems throughout 2026.
  • The AI supply chain is more fragile than we think. One developer forgot to exclude a source map from a production build, and the entire source code of one of the most popular AI tools on Earth was exposed. The AI industry moves fast, but its security practices haven't caught up.

What's Next

The Claude Code leak will have lasting effects. Security researchers are already analyzing the anti-cloning mechanisms, and Anthropic will need to fundamentally rethink how it ships its tools. The company raised $30 billion in February 2026 at a $380 billion valuation — the stakes for getting this right couldn't be higher.

For Google, Gemma 4 is a shot across the bow of Meta's Llama family. With Meta reportedly shifting toward closed-source models and delaying its Llama 4 successors (Mango and Avocado), Google has an opportunity to become the default open-source AI provider. The Apache 2.0 license makes that possible.

For developers, the lesson is clear: the tools you depend on might be more fragile — and more proprietary — than they appear. And the best models for local, private, unrestricted use are increasingly coming from companies willing to truly open-source them.

This was the week that reminded us that in AI, the biggest disruptions don't always come from press releases. Sometimes they come from a forgotten file in an npm package — and sometimes from a deliberate decision to give everything away.
