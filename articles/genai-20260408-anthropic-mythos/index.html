Anthropic's Claude Mythos: The AI Model Too Dangerous to Release

A frontier AI model that finds zero-days in every major OS and browser. 72.4% exploit success rate. And the capabilities weren't even explicitly trained.

April 8, 2026 · AI Safety / Cybersecurity

Claude Mythos Preview: the frontier model that changed cybersecurity forever

On April 7, 2026, Anthropic made an announcement that sent shockwaves through the cybersecurity world. They revealed Claude Mythos Preview — a general-purpose frontier AI model so capable at finding and exploiting software vulnerabilities that they decided not to release it to the public.

Instead, they launched Project Glasswing, a coalition with AWS, Apple, Google, Microsoft, Cisco, CrowdStrike, NVIDIA, and others to use the model defensively. The story behind Mythos is about far more than one model. It's about a fundamental shift in the relationship between AI and cybersecurity — and the uncomfortable truth that these capabilities emerged without anyone specifically training for them.

What is Claude Mythos Preview?

Mythos Preview is a general-purpose language model trained by Anthropic. It performs well across standard benchmarks, but its cybersecurity capabilities are what made Anthropic hit the brakes on a public release.

According to Anthropic's own assessment published on their red team blog:

"AI models have reached a level of coding capability where they can surpass all but the most skilled humans at finding and exploiting software vulnerabilities."

This isn't marketing language. The technical details back it up with hard numbers.

The Numbers: Opus 4.6 vs Mythos

Mythos Preview vs Claude Opus 4.6 on key cybersecurity benchmarks (Source: Anthropic Red Team)

The leap from Anthropic's previous model is staggering:

  • Firefox JavaScript exploit development: Claude Opus 4.6 managed to turn discovered vulnerabilities into working shell exploits only 2 times out of several hundred attempts (near 0%). Mythos Preview? 181 working exploits out of roughly 250 attempts — a 72.4% success rate.
  • OSS-Fuzz benchmark (7,000 entry points): Opus 4.6 and Sonnet 4.6 each achieved only a single crash at Tier 3 severity. Mythos Preview achieved 595 crashes at Tiers 1-2, a handful at Tiers 3-4, and 10 full control flow hijacks at Tier 5 — the highest severity level.
  • Zero-day discovery: Mythos found zero-day vulnerabilities in every major operating system and every major web browser.

For context on how fast this happened: just one month earlier, Anthropic's own researchers wrote that "Opus 4.6 is currently far better at identifying and fixing vulnerabilities than at exploiting them." Mythos Preview flipped that equation entirely.

What Can It Actually Do?

The exploit examples from Anthropic's assessment read like a security researcher's nightmare:

  • Browser sandbox escape: Mythos wrote a web browser exploit that chained together four separate vulnerabilities, using a JIT heap spray technique to escape both the renderer and the operating system sandboxes. This is the kind of multi-vulnerability chain that top-tier exploit developers spend months building.
  • Linux privilege escalation: It autonomously developed local privilege escalation exploits by exploiting subtle race conditions and KASLR (Kernel Address Space Layout Randomization) bypasses.
  • FreeBSD NFS root exploit: Mythos wrote a remote code execution exploit for FreeBSD's NFS server that granted full root access to unauthenticated users — by splitting a 20-gadget ROP (Return-Oriented Programming) chain across multiple network packets.
  • Ancient bug discovery: It found a 27-year-old bug in OpenBSD — an operating system famous for its obsessive focus on security. Many of the vulnerabilities it finds are 10-20 years old, hiding in plain sight.

Perhaps most unsettling: Anthropic engineers with no formal security training asked Mythos to find remote code execution vulnerabilities overnight and woke up to complete, working exploits. You don't need to be a security expert anymore. The model handles the expertise.

The Emergent Capability Problem

Anthropic did not explicitly train Mythos for cybersecurity. These capabilities emerged as a "downstream consequence of general improvements in code, reasoning, and autonomy." The same improvements that make the model better at patching bugs also make it better at exploiting them. This raises a fundamental question: if security capabilities emerge unsolicited from general training, how do we prevent them in future models?

Project Glasswing: The Defensive Response

Project Glasswing: Anthropic's $100M+ defensive coalition

Instead of releasing Mythos publicly — or shelving it entirely — Anthropic chose a middle path. Project Glasswing gives controlled access to the model for defensive purposes:

  • Launch partners: AWS, Apple, Google, Microsoft, Cisco, CrowdStrike, NVIDIA, Broadcom, Palo Alto Networks, JPMorganChase, and the Linux Foundation
  • Extended access: Over 40 additional organizations that build or maintain critical software infrastructure
  • Financial commitment: Up to $100M in Mythos Preview usage credits, plus $4M in direct donations to open-source security organizations

The idea is straightforward: use Mythos to find and fix vulnerabilities before malicious actors can exploit them. It's a race between defenders and attackers, and Anthropic is giving defenders a head start.

The Timeline: How We Got Here

The road from leak to official announcement

Mythos didn't arrive out of nowhere. The story unfolded over two weeks:

  • March 27: Mythos documentation leaked via a public CMS — approximately 3,000 files. Cybersecurity stocks dropped sharply on the news. CNBC, CoinDesk, and other outlets reported on the leaked details.
  • March 29: Axios ran a piece titled "AI's looming cyber nightmare." Analysis of the leak was published on Medium. The industry started paying serious attention.
  • April 3: CNN reported that Mythos could be a "watershed moment" for cybersecurity.
  • April 7: Anthropic made it official. The Glasswing announcement (1,396 HN points, 716 comments), the Mythos system card, and the detailed red team assessment were all published simultaneously. Bloomberg reported that Apple and Amazon were given early access. The New York Times called it a "cybersecurity reckoning."
  • April 8: Coverage continued from Euronews, CNN, The Register, and Platformer. Cybersecurity experts weighed in across the board.

What This Means for the Industry

The ripple effects of Mythos across the tech industry

For Open Source Maintainers

Open-source projects — already under-resourced and struggling with security — face a new reality. The good news: Project Glasswing extends access to open-source security organizations. The bad news: if Anthropic's model can find these bugs, competing models from less scrupulous actors will too, and probably sooner than anyone wants to admit. Open-source maintainers need more support, funding, and tooling — not less.

For Cybersecurity

Anthropic draws a parallel to the history of software fuzzers. When fuzzers like AFL were first deployed at scale, there were concerns they'd benefit attackers more than defenders. Instead, they became critical defensive tools (OSS-Fuzz, for example). Anthropic believes the same will happen with AI security models — eventually. The dangerous period is the transition: when these capabilities exist but the defensive infrastructure isn't mature enough to keep up.

For AI Safety

This is perhaps the most alarming aspect. Mythos wasn't trained to hack. Its security capabilities emerged from general improvements in reasoning, code understanding, and autonomy. If frontier models naturally develop dangerous capabilities as a side effect of getting better at normal tasks, the entire alignment problem becomes much harder. You can't easily remove a capability that wasn't explicitly added.

For Developers

Every software developer needs to take security more seriously. "Shift-left security" — integrating security testing early in the development lifecycle — is no longer optional. If an AI can find exploitable bugs in your code overnight, you want to find them first.

For the Broader Industry

Global cybercrime costs an estimated $500 billion per year. That number is likely to grow before it shrinks. Cybersecurity stocks were already volatile on the Mythos leak news in March. Expect new AI-powered security startups to emerge rapidly, and expect patch cycles to accelerate dramatically.

The Uncomfortable Questions

Mythos raises questions that don't have easy answers:

  • If Anthropic can build this, who else already has? State actors like China, Russia, and North Korea invest heavily in both AI and offensive cyber capabilities. It would be naive to assume they're not pursuing similar research.
  • Can you put the genie back in the bottle? Mythos proves that these capabilities emerge naturally from scaling. Even if Anthropic doesn't release it, the next frontier model from any major lab may develop similar capabilities independently.
  • Is the "responsible disclosure" model enough? Over 99% of the vulnerabilities Mythos found remain unpatched. Coordinated disclosure takes time. The window between discovery and patch is a window of vulnerability.
  • What happens when these models get cheaper to run? Today, Mythos-level capability requires significant compute. In a year or two, it might run on consumer hardware. Then what?

Lessons for Builders

If you're a developer or building software, here's what I'm taking away from this:

  1. Assume your code has vulnerabilities. If a model can find 0-days in OpenBSD, it can find them in your project. Use static analysis, fuzzing, and AI-assisted security tools now.
  2. Keep dependencies updated. Many of Mythos's exploits target known-but-unpatched (N-day) vulnerabilities. Patching fast is one of the most effective defenses.
  3. Treat security as a first-class concern. Not a quarterly review. Not an afterthought. If AI can exploit your codebase overnight, you need continuous security monitoring.
  4. Watch the open-source projects you depend on. The $4M Anthropic is donating to open-source security organizations is a start, but the real solution is sustainable funding for critical open-source infrastructure.

What's Next

Project Glasswing is described as a starting point, not a solution. Anthropic's own words: "The work of defending the world's cyber infrastructure might take years; frontier AI capabilities are likely to advance substantially over just the next few months."

That timeline mismatch — years of defensive work needed, months of capability advancement — is the core challenge. The next frontier model could be even more capable than Mythos. The question isn't whether AI will transform cybersecurity. It already has. The question is whether we'll be ready when it does.

For now, over 99% of Mythos's discovered vulnerabilities remain unpatched. The clock is ticking.
